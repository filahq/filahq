# FilaHQ Landing Page Design

**Date:** 2026-04-11

## Goal

Create a single-page Astro landing page for FilaHQ that presents Statify as the flagship product, positions FilaHQ as the studio behind it, and explains the current cross-platform Statify ecosystem for Filament developers.

## Audience

Filament developers who want to share Filament stats beyond the admin panel and access them across platforms and devices.

## Messaging Direction

- Lead with `Statify` as the flagship.
- Present `FilaHQ` as the studio behind the product line.
- Explain the ecosystem as a workflow:
  `Statify Easy Widget` helps build stats widgets,
  `Statify` exposes them as secure endpoints,
  `Statify Extension` surfaces them in Chrome.
- Reinforce that more integrations are coming for desktop, tablet, mobile, and other surfaces.

## Page Structure

1. **Hero**
   - Headline centered on Statify's value.
   - FilaHQ shown as a supporting studio label.
   - Three primary actions:
     - Install Statify via Composer
     - Install Statify Easy Widget via Composer
     - Add Statify Extension from Chrome Web Store

2. **Workflow**
   - Three-step narrative:
     - Build widgets faster with Statify Easy Widget
     - Expose them securely with Statify
     - View them in Chrome with Statify Extension

3. **Cross-Platform Momentum**
   - Forward-looking section that hints at upcoming desktop, tablet, mobile, and broader integrations.

4. **Product Detail**
   - One focused block per current product:
     - Statify
     - Statify Easy Widget
     - Statify Extension

5. **Closing CTA**
   - Repeat the install/add actions and reinforce the product ecosystem.

6. **Footer**
   - Minimal footer with FilaHQ studio line.
   - Links to product destinations.
   - Quiet note about more integrations coming.

## Visual Direction

- Primary palette:
  - Accent: `#ffcc00`
  - Base dark: `#343434`
- Visual mood:
  bold, engineering-focused, poster-like, high contrast.
- Layout:
  full-bleed hero, restrained sections, no dashboard-card overload.
- Imagery:
  abstract cross-device surfaces and data rails instead of fake UI screenshots.
- Typography:
  bold and direct, with strong hierarchy around the Statify name.

## Motion

- Short hero entrance sequence.
- Scroll-based reveal for workflow and integration sections.
- Subtle depth shifts for device and surface hints.

## Content Notes

- Mention current status:
  - `Statify` supports Filament 4 and 5 and latest pushed release `v0.2.0`.
  - `Statify Easy Widget` supports Filament 4 and 5 and latest pushed release `v0.2.0`.
  - `Statify Extension` connects secure HTTPS Statify endpoints and shows dashboard widgets in one Chrome tab.
- Keep copy concise and product-oriented.
- Avoid generic SaaS sections such as testimonials, logo walls, or oversized footers.
